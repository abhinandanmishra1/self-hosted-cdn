BlinkCDN is an open-source project, and we welcome contributions from the community. Whether you're fixing a bug, improving the documentation, or adding a new feature, your help is appreciated.

## How to Contribute

### 1. Fork the Repository
Click the "Fork" button at the top right of the repository page to create your own copy of the project.

### 2. Clone Your Fork
Clone your forked repository to your local machine:
```bash
git clone https://github.com/your-username/blinkcdn.git
cd blinkcdn
```

### 3. Create a Branch
Create a new branch for your changes:
```bash
git checkout -b feature/my-new-feature
# or
git checkout -b fix/my-bug-fix
```

### 4. Install Dependencies
Install the required dependencies:
```bash
npm install
```

### 5. Make Your Changes
Make your changes to the code. Ensure that your code follows the project's coding standards.

### 6. Test Your Changes
Run the development server to test your changes:
```bash
npm run dev
```
Open `http://localhost:3000` in your browser.

### 7. Commit Your Changes
Commit your changes with a descriptive commit message:
```bash
git commit -m "feat: add amazing new feature"
```

### 8. Push to GitHub
Push your changes to your forked repository:
```bash
git push origin feature/my-new-feature
```

### 9. Submit a Pull Request
Go to the original repository and click the "New Pull Request" button. Select your branch and submit your pull request.

## Development Guidelines

*   **Code Style**: Keep the code clean and consistent.
*   **Commits**: Use clear and concise commit messages.
*   **Documentation**: Update documentation if you change functionality.

## Reporting Bugs
If you find a bug, please create an issue on GitHub describing the problem and how to reproduce it.

Thank you for contributing!
